Repro of <https://github.com/uber/NullAway/issues/1437> for convenience.

There appears to be a difference in the `JSpecifyMode` option's behaviour between Maven and Gradle.
The code contains a bug that should cause NullAway to fail the build.
Maven incorrectly passes whereas Gradle fails when `JSpecifyMode=true` and passes when `JSpecifyMode=false`.
Maven's behaviour has been observed in JDK 21 and JDK 25, with different JDK distributions, on Linux and Windows, and with and without forking.
The options `AnnotatedPackages` and `OnlyNullMarked` are interchangeable here.
